# Cron Design

## Design Goal
Support repeated checks during the monitoring window without generating duplicate reports.

## Conceptual Jobs
### Job 1: Open Window
- Marks the market day monitoring as started
- Initializes daily state if needed

### Job 2: Repeated Check
- Runs every 30 minutes during the monitoring window
- Checks state first
- If report already generated or monitoring completed, exit quietly
- Otherwise check the channel for a same-day video
- If valid, generate the report and mark completed

### Job 3: Close Window
- Marks monitoring as completed if no report was generated by deadline
- Records stopReason = no_same_day_video_by_deadline

## Important Behavior
- Repeated checks must be idempotent
- Close-window logic must not overwrite a successful report state
- The system must treat late-night checks as belonging to the same market day being monitored
