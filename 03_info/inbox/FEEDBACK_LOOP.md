# Email Feedback Loop

## Purpose
Continuously improve email triage quality.

## Inputs to Learn From
- false positives (important mail wrongly downgraded)
- false negatives (junk mail left in Inbox)
- repetitive good senders
- repetitive bad senders
- changing business context

## Update Targets
- sender score
- keyword weights
- category logic
- routing thresholds

## Rule
The system should evolve from real usage, not rely forever on initial assumptions.
